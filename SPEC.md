# SPEC — VS MOTORS

Telegram Mini App — личная витрина автомобилей одного продавца.
Один владелец, никакой регистрации и монетизации.

---

## Концепция

**VS MOTORS** — это Telegram Mini App, которое работает как витрина автомобилей одного продавца.
Владелец добавляет машины через закрытую админку прямо в приложении.
Пользователи листают витрину, смотрят детали и связываются с владельцем одним касанием.

**Целевая аудитория:** покупатели автомобилей, которые уже в Telegram.
**Язык интерфейса:** русский (один язык, без i18n).
**Монетизация:** отсутствует.
**Мультипользовательские аккаунты:** отсутствуют (один владелец = один пароль).

## Страницы приложения

### 1. HomePage — Витрина автомобилей

**Маршрут:** `/`

**Элементы:**
- Шапка: логотип VS MOTORS
- Карточки автомобилей (бесконечная прокрутка, PAGE_SIZE=20)
- Кнопка «Добавить» — только если пользователь авторизован как владелец (сравнение Telegram user_id с OWNER_ID из env)
- Пустое состояние: «Объявлений пока нет» если витрина пуста

**Карточка автомобиля:**
- Главное фото (соотношение 16:9, fallback — placeholder)
- Марка + Модель + Год (заголовок)
- Цена (жирно, в USD или ILS — настраивается в ENV)
- Пробег (км)
- Значки: КПП (авто/механика), тип топлива
- Статус: «Продаётся» / «Продан» (если продан — карточка полупрозрачная)

**Сортировка:** по умолчанию — новые сверху (по дате добавления).

- Объём двигателя (л, число)

- Привод (select: Передний / Задний / Полный)

## Telegram Bot

**Имя бота:** задаётся в ENV: `BOT_USERNAME`

**Команды:**
- `/start` — приветствие + кнопка «Открыть VS MOTORS» (открывает Mini App)
- `/help` — справка: что такое бот, как смотреть объявления

**Deep links:**
- `https://t.me/{BOT_USERNAME}?startapp=listing_{id}` — открывает Mini App сразу на странице конкретного автомобиля
- При получении `startapp` параметра — редирект внутри Mini App на `/listing/{id}`

**Кнопка «Открыть VS MOTORS»:** тип `web_app`, URL — MINI_APP_URL из ENV.

## Авторизация владельца

Владелец — единственный привилегированный пользователь.

**Способ идентификации:** сравнение `telegram_id` из `initData` с `OWNER_TELEGRAM_ID` из ENV.

**Логика на бэкенде:**
```
if user.telegram_id == settings.OWNER_TELEGRAM_ID:
    user.is_owner = True
```

**Никакого пароля, никакого JWT для обычных пользователей.** Обычные пользователи не авторизуются — они просто смотрят.

**Владелец получает:**
- Кнопку «Добавить» на HomePage
- Кнопки «Редактировать» / «Удалить» на ListingDetailPage
- Доступ к AdminPage и AddListingPage
- `/admin` маршрут

## API эндпоинты

**Base URL:** задаётся в ENV `API_URL`

### Публичные (без авторизации)

| Метод | Путь | Описание |
|---|---|---|
| GET | `/health` | Healthcheck: `{"status": "ok", "db": "ok"}` |
| GET | `/api/listings` | Список активных объявлений с пагинацией |
| GET | `/api/listings/{id}` | Детали одного объявления |

**Параметры GET /api/listings:**
- `page` (int, default=1)
- `page_size` (int, default=20, max=50)

Фильтрация отсутствует — витрина показывает все активные объявления.

### Владелец (требует `X-Telegram-Init-Data` заголовок)

| Метод | Путь | Описание |
|---|---|---|
| POST | `/api/listings` | Создать объявление |
| PATCH | `/api/listings/{id}` | Обновить объявление |
| DELETE | `/api/listings/{id}` | Удалить объявление |
| POST | `/api/listings/{id}/photos` | Загрузить фото (подпись Cloudinary) |
| DELETE | `/api/listings/{id}/photos` | Удалить фото из массива (URL в body) |
| GET | `/api/admin/listings` | Все объявления включая «Продан» |

**Верификация владельца на бэкенде:**
1. Извлечь `telegram_id` из `X-Telegram-Init-Data` (верифицировать HMAC с `BOT_TOKEN`)
2. Сравнить с `OWNER_TELEGRAM_ID` из ENV
3. Если не совпадает — `403 Forbidden`

## Загрузка фото

**Сервис:** Cloudinary

**Флоу:**
1. Фронтенд вызывает `POST /api/listings/{id}/photos` → бэкенд генерирует signed upload parameters
2. Фронтенд грузит файл напрямую на Cloudinary (не через бэкенд)
3. Cloudinary возвращает URL → фронтенд добавляет в список фото объявления
4. При сохранении объявления — массив URL записывается в `listings.photos`

**Ограничения:**
- Максимум 20 фото на объявление
- Максимальный размер файла: 10 MB
- Поддерживаемые форматы: JPEG, PNG, WEBP, HEIC
- Cloudinary автоматически конвертирует в WEBP при отдаче
- Трансформация: `w_1200,h_675,c_fill,q_auto` (16:9, 1200px ширина)

## Технологический стек

### Frontend
- **React 18** + **Vite** + **TypeScript**
- **React Router v6** (навигация)
- **@twa-dev/sdk** (Telegram Mini App SDK)
- Без i18n (один язык — русский)
- Тёмная/светлая тема из `Telegram.colorScheme`
- CSS variables для цветов темы

### Backend
- **FastAPI** + **aiogram 3**
- **asyncpg** (`prepared_statement_cache_size=0` — обязательно для Supabase Pooler)
- Один процесс: FastAPI + бот через `lifespan`

### Инфраструктура
- **Render** (бэкенд, Free tier, keep-alive через UptimeRobot)
- **Vercel** (фронтенд)
- **Supabase** (PostgreSQL)
- **Cloudinary** (фото, Free tier)

## Ограничения и допущения

- Без фильтров и поиска — это витрина, не каталог с поиском
- Без регистрации и авторизации пользователей
- Без избранного (нет аккаунтов)
- Без чата внутри приложения (только внешние контакты: TG, WA, телефон)
- Без push-уведомлений пользователям
- Без статистики просмотров (возможно добавить в v2)
- Одна валюта по умолчанию, но каждое объявление хранит свою
